Virtual Machines

Le macchine virtuali sono nate per rispondere all’esigenza di eseguire più sistemi operativi sullo stesso hardware fisico. Scopri quando sono state introdotte, come funzionano attraverso la virtualizzazione delle risorse, quali tipologie principali esistono e come strumenti come i cgroups permettono di gestire e isolare l’uso delle risorse in modo efficiente.

Quando si utilizzano le Macchine Virtuali?

Le macchine virtuali vengono utilizzate ogni volta che è necessario creare un ambiente sicuro, isolato e flessibile rispetto al computer reale. Sono molto utili quando si vuole provare un nuovo sistema operativo, come Linux o una versione diversa di Windows, senza doverlo installare direttamente sul proprio dispositivo. In questo modo si possono fare esperimenti, modifiche o errori senza correre il rischio di danneggiare il sistema principale.

  • Un altro momento in cui si usano le macchine virtuali è quando serve maggiore sicurezza. Poiché la macchina virtuale è isolata dal computer fisico, eventuali virus, file sospetti o errori rimangono confinati al suo interno. Questo le rende ideali per analizzare malware, aprire allegati non sicuri o separare diversi ambienti di lavoro
  • Le VM sono molto utili anche per chi sviluppa software. Permettono di testare un programma su più sistemi operativi e configurazioni, verificando che funzioni correttamente ovunque. Inoltre, consentono a più persone di lavorare sullo stesso progetto usando un ambiente identico, evitando problemi di compatibilità.
  • Un altro caso importante è quando si vogliono risparmiare risorse hardware. In un solo computer potente si possono creare più macchine virtuali, invece di usare più computer fisici. Questo è molto diffuso nelle aziende, nei data center e nei servizi cloud, perché permette di ridurre costi, consumi di energia e spazio.
  • Infine, si utilizzano quando è necessario eseguire software vecchio o non più compatibile con i sistemi moderni. Grazie alle VM è possibile ricreare un computer “del passato” e far funzionare programmi che altrimenti non potrebbero più essere usati.

Cosa sono e come funzionano

Una VM è una macchina simulata all'interno di una macchina fisica, che sfrutta come risorse virtuali la CPU, la RAM e lo Storage (HDD o SSD) in modo da eseguire il Sistema Operarivo in modo isolato

L’Hypervisor: Il Gestore delle Risorse

L’hypervisor è il componente chiave che: Alloca le risorse (CPU, memoria, storage) alle VM. Garantisce l'isolamento tra le diverse VM. Può essere di due tipi:

(Bare-metal): Gira direttamente sull’hardware (es. VMware ESXi, Microsoft Hyper-V).

(Hosted): Gira su un sistema operativo esistente (es. VirtualBox, VMware Workstation).

Risorse Virtuali: Come Funziona una VM

Ogni VM ha le sue risorse virtuali: CPU virtuale (vCPU) Memoria virtuale (vRAM) Disco virtuale (file come VMDK, VHD) Scheda di rete virtuale Queste risorse sono mappate su quelle fisiche, ma la VM le utilizza come se fossero dedicate.

Sistema Operativo Guest

All’interno della VM gira un sistema operativo guest (Windows, Linux, ecc.),completamente indipendente dall’host e dalle altre VM. Questo consente di:

Eseguire software non compatibile con l’host. Testare nuovi sistemi operativi in sicurezza.

Limitazioni

Overhead: La virtualizzazione consuma risorse aggiuntive.

Prestazioni: Una VM può essere meno performante di un sistema fisico dedicato.

Passtrought

GPU Passthrough è una tecnica che permette a una macchina virtuale di usare direttamente una scheda grafica fisica, con prestazioni quasi identiche a quelle reali. Si basa su tecnologie come IOMMU e VFIO, che isolano la GPU dal sistema host e la assegnano alla VM. È utile per il gaming o il rendering, perché la VM può sfruttare la GPU come se fosse installata direttamente.

Quali tipi di macchine virtuali esistono

Virtual Machine System

Simulano un intero computer fisico e permettono di eseguire un sistema operativo completo. Esempi: KVM (Linux) VMware ESXi / Workstation / Fusion Microsoft Hyper-V VirtualBox Sono utilizzate per: Server virtualizzati Testing di sistemi operativi Isolamento applicativo Cloud computing (AWS, Azure, GCP)

Virtual Machine di processo (Process VM)

Eseguono una singola applicazione all’interno di un ambiente virtualizzato, senza simulare un intero sistema operativo. Esempi:

Java Virtual Machine (JVM) .NET CLR WebAssembly runtimes Sono usate per: Portabilità delle applicazioni Isolamento del codice Esecuzione sicura

Container (simili ma non VM)

Non sono VM vere e proprie, ma vengono spesso citati assieme. Condividono il kernel dell’host e isolano solo i processi. Esempi: Docker LXC/LXD Kubernetes (orchestrazione)

Cosa Sono i CGroups?

I CGroups (Control Groups) sono una funzionalità del kernel Linux che permettono di limitare, monitorare e isolare l'uso delle risorse (CPU, memoria, I/O su disco, rete, ecc.) tra diversi gruppi di processi.

In breve, consentono di controllare quante risorse può consumare un processo o un gruppo di processi. Sono alla base di tecnologie come Docker, LXC, Kubernetes e systemd.

  • Limitano l'utilizzo di risorse (CPU, RAM, I/O).
  • Permettono il monitoraggio in tempo reale.
  • Isolano gruppi di processi per stabilità e sicurezza.
  • Permettono pausa/ripresa e prioritizzazione.
Versioni

CGroups v1: Introdotta nel kernel 2.6.24, con controller separati per tipo di risorsa.

CGroups v2: Versione moderna con gerarchia unificata e gestione più semplice.

Utilizzo con VM e Container

I CGroups operano a livello del sistema operativo e possono gestire le risorse delle macchine virtuali sull'host o dei container all'interno di una VM.

Approfondimento Tecnico

I Control Groups implementano un modello gerarchico nel quale i processi possono essere organizzati in diversi gruppi, ciascuno sottoposto a specifiche politiche di gestione delle risorse. Il kernel interagisce con i controller per applicare limiti, raccogliere statistiche e modulare la schedulazione.
